# 轻语（Whisper）多Agent小说写作助手  
**产品需求规格说明书（大纲）**

---

## 1. 引言
### 1.1 项目背景  
- 快节奏时代对网文/小说的需求量增大  
- AI 辅助创作可提升效率与灵感，降低创作门槛  

### 1.2 产品愿景  
- 成为小说作家案头最得力的 AI 协作者，覆盖从灵感到发布的全流程  

### 1.3 目标用户  
- 有抱负的新手作家、寻求效率突破的成熟作者、网文工作室  

### 1.4 核心价值主张  
- 以“构思-计划-写作-修改-发布”真实创作流为骨架，多 Agent 配合专业工具，让 AI 真正融入写作，而非简单问答  

---

## 2. 产品概览
### 2.1 产品形态  
- **无登录桌面端**：本地优先，单机使用，注重隐私与极简体验  
- **有登录完整版**：增加云同步、分享、等级、发布等业务能力  

### 2.2 核心工作流（真实作家流程映射）  
构思 → 计划（大纲/分幕） → 写作（续写/扩写） → 修改/编辑 → 发布/完善
每个阶段均可由 AI 辅助，支持循环跳转。

### 2.3 关键特性摘要  
- 双模型协同（DeepSeek V4 Pro 思考 + Flash 写作）  
- 可配置的技能系统与 MCP 外部工具扩展  
- 多 Agent 角色扮演剧场，并内置 token 爆炸缓解策略  
- 结构化设定文件管理与关系可视化  
- 完整离线可用、隐私第一的无登录模式  

---

## 3. 用户角色与典型场景
### 3.1 用户角色定义  
- **纯本地创作者**：无需登录，所有数据本地  
- **联网同步作者**：多设备同步，分享作品，享受增值服务  
- **工作室管理者**：管理共同设定库，协作编辑  

### 3.2 典型使用场景  
- 新人作者用“构思”生成故事梗概，通过“计划”得到章纲，再逐章写作  
- 资深作者在修改阶段让 AI 检测一致性、语气调整，并通过角色剧场模拟对话  
- 工作室统一设定文件，多名作者基于相同世界观协同创作  

---

## 4. 功能性需求
### 4.1 基础聊天与模型交互  
#### 4.1.1 多模型 API 接入  
- 支持本地部署及第三方（如 DeepSeek）API 地址配置  
- 可为目标任务指定模型（如思考用 V4 Pro，写作流用 Flash）  
#### 4.1.2 流式通信（SSE）  
- 实时打字机效果，可随时中断生成  
#### 4.1.3 对话管理  
- 新建/切换会话，会话历史本地持久化，支持关键词搜索  

### 4.2 智能循环控制（原“API 请求循环”明确化）  
#### 4.2.1 自动迭代回路  
- 用户可启用“深度思考”模式，系统自动多次调用模型，反思并优化结果，直至满足预定条件或用户中止  
- 典型应用：大纲不满意时自动重构、情节矛盾自动修正  
#### 4.2.2 用户干预  
- 实时显示循环状态，支持“停止”“采用当前结果”“手动调整后继续”  

### 4.3 技能系统（可配置的写作增强套件）  
#### 4.3.1 技能定义  
- 技能 = 专属 System Prompt 片段 + 可调用工具列表 + 触发场景说明  
- 示例：“古风言情技能”会注入古风用词约束，并可调用诗词生成工具  
#### 4.3.2 技能市场  
- 提供技能搜索、浏览、一键下载/更新，支持用户自定义技能并本地保存  
#### 4.3.3 技能调用  
- 对话中可 @技能名 强制启用；写作阶段可自动激活预配技能  
#### 4.3.4 强制模式  
- 当用户指定技能时，模型必须遵循该技能设定，不得忽略  

### 4.4 MCP 服务集成  
#### 4.4.1 MCP 客户端  
- 内置在桌面应用中，负责发现、连接、管理 MCP 服务  
#### 4.4.2 服务连接与工具注册  
- 支持输入 MCP 服务地址进行连接，通过 OAuth 授权  
- 成功连接后，服务提供的工具自动进入“工具系统”，可被技能和大模型调用  
#### 4.4.3 权限与安全  
- 调用外部工具前展示工具描述与所需权限，用户确认后执行；支持连接日志和工具调用频率限制  

### 4.5 多 Agent 角色扮演（含 Token 爆炸缓解）  
#### 4.5.1 角色 Agent 定义  
- 每个 Agent 有独立设定集（性格、语言风格、背景知识、目标）  
#### 4.5.2 同框剧场模式  
- **观察模式**：多个 AI 角色自动互动推进剧情，用户旁观  
- **参与模式**：用户扮演一个角色，与其他 AI 角色对话  
- 支持“导演指令”插入，调整角色行为  
#### 4.5.3 Token 爆炸应对策略（必选）  
- 上下文窗口动态管理（旧对话摘要压缩、滑动窗口）  
- 角色记忆外置：将长期记忆存入向量数据库，按需检索  
- 设定摘要注入：仅将压缩后的角色关键信息放入 prompt  
- 后台动态装载/卸载非活跃角色  
#### 4.5.4 会话调度  
- 系统决定发言顺序，支持“抢话”开关，防止死循环  

### 4.6 写作全流程支持  
#### 4.6.1 构思阶段  
- 头脑风暴：根据关键词生成选题、冲突设定、高概念梗概  
- “如果…会怎样”发散问答  
#### 4.6.2 计划阶段  
- 生成多级章节大纲，可视化拖拽调整结构  
- 分幕节奏建议、人物弧光规划、伏笔埋设提示  
#### 4.6.3 写作阶段  
- 逐段/逐章续写，支持风格模仿、多方案生成备选  
- 实时字数统计，目标进度条  
#### 4.6.4 修改/编辑阶段  
- 语法与错别字校正、语气调整、一致性检查（人名/地名/设定冲突）  
- 敏感内容提示，扩写/缩写段落  
#### 4.6.5 发布/完善阶段  
- 一键导出（TXT/Markdown/DOCX）  
- 生成作品简介、推荐语、封面图 Prompt，可对接图片生成工具  
- （登录版）支持一键发布至合作平台或生成分享链接  

### 4.7 工具系统  
#### 4.7.1 内置工具清单  
- **读取**：当前草稿、指定章节、设定文件、笔记  
- **写入**：新增章节、更新设定、插入片段  
- **编辑**：重写、润色、调整语调、替换词语  
- **搜索**：联网参考资料检索（可限定来源）  
- **图片生成**：角色立绘、场景图、封面（基于设定描述）  
- **大纲工具**：添加/移动/删除章节节点  
- **检测工具**：敏感词过滤、一致性检查  
#### 4.7.2 调用流程与可视化  
- 模型发出工具调用请求 → 用户界面展示工具名及参数 → 用户确认（可设置自动执行） → 执行并返回结果  
#### 4.7.3 联网与生成约束  
- 联网搜索有频次限制，需用户显式许可  
- 图片生成结果本地缓存，且关联到对应设定或章节  

### 4.8 设定文件管理  
#### 4.8.1 设定卡类型  
- 人物、势力、世界/地点、物品、技能/魔法体系、历史事件  
- 每类卡有预设模板字段，支持自定义扩展  
#### 4.8.2 智能采集与编辑  
- AI 可从对话/草稿中自动提取新设定项，提示用户确认存入  
- 提供富文本编辑器，支持相互引用（如人物卡里选择所属势力）  
#### 4.8.3 版本历史  
- 每张设定卡保留修改记录，可回滚到历史版本  
#### 4.8.4 关系图谱  
- 自动生成人物关系图、势力树，支持手动调整和筛选  
#### 4.8.5 导入/导出  
- 支持 JSON/YAML 格式，便于备份和跨项目复用  

### 4.9 用户与内容管理（有登录版）  
#### 4.9.1 账号认证  
- JWT 登录，支持邮箱/手机注册  
#### 4.9.2 等级与权益  
- 免费用户：每日生成字数上限、基础模型，部分技能限制  
- 高级用户：更大上下文、优先队列、专属技能、图片生成额度  
#### 4.9.3 云存储与同步  
- 作品、设定、会话自动同步，冲突时提供版本合并提示  
#### 4.9.4 分享与协作  
- 生成只读分享链接，支持密码保护；可邀请协作者共同编辑  
#### 4.9.5 发布服务  
- 格式化后直接推送至目标创作平台（需对接 API），或导出适配格式  

### 4.10 本地离线支持（无登录版）  
#### 4.10.1 本地项目  
- 以文件夹形式组织，数据存于 SQLite 或文件系统，全离线可用  
#### 4.10.2 备份与恢复  
- 手动/自动备份为压缩包，支持恢复指定版本

## 5. 非功能性需求
### 5.1 性能  
- 流式首字响应时间 < 1.5 秒（视网络/模型部署而定）  
- 10 万字草稿打开与编辑不卡顿  
### 5.2 可用性  
- 新用户首次引导（示例项目）；所有功能有明确帮助入口  
- 支持深色/浅色主题，专注写作模式  
### 5.3 安全与隐私  
- 无登录版数据绝不上传；联网搜索等需显式确认  
- 云同步数据端到端加密，遵守隐私法规  
### 5.4 可扩展性  
- 模型驱动可插拔，技能和 MCP 工具遵循统一接口  
- 前端组件化，便于社区开发主题和插件  
### 5.5 兼容性  
- 桌面端支持 Windows/macOS（若为 Web 技术封装，需明确 Electron 或 Tauri）

## 6. 技术架构（概要）
### 6.1 整体架构  
- 表现层（React + TypeScript） → 服务层（对话状态管理、技能调度、MCP 客户端） → API 层（模型路由、工具执行） → 数据层  
### 6.2 前端技术栈  
- React + TypeScript，UI 库与状态管理（如 Zustand）  
### 6.3 后端技术栈（有登录版）  
- SpringBoot + JWT，Redis 用于会话缓存，消息队列处理异步任务  
### 6.4 模型路由  
- 智能分发：思考/规划类任务→V4 Pro，写作/续写→Flash，可降级和超时切换  
### 6.5 对话状态管理  
- 每个会话独立维护：消息历史、激活技能、已加载角色、写作阶段、工具调用上下文  
- 支持序列化到本地或云端  
### 6.6 MCP 集成方案  
- 桌面端内嵌 MCP 客户端进程，通过标准协议与外部服务通信  

---

## 7. 界面与交互设计原则
### 7.1 布局框架  
- 三栏可调节：左侧资源树（大纲/设定/文件），中间编辑/对话区，右侧动态面板（属性/工具/技能）  
### 7.2 视觉风格  
- 清新、留白，衬线字体可选，营造沉浸式阅读与写作氛围  
### 7.3 关键交互  
- 角色扮演消息气泡区分（头像+颜色+名称），生成中断按钮明显，流式文本平滑出现  

---

## 8. 数据模型设计（概略）
- **Project**：作品集，包含设定库、章节、会话  
- **Chapter**：标题、内容、状态、字数  
- **SettingCard**：类型、字段 JSON、历史版本  
- **Skill**：名称、描述、Prompt 模板、关联工具  
- **AgentRole**：名称、系统 Prompt、头像、记忆上下文指针  
- **User（登录版）**：账户、等级、配额、作品列表  

---

## 9. 项目规划与里程碑
### 9.1 MVP（最小可行产品）  
- 无登录版，支持基础聊天、构思→写作→修改流程、设定卡管理、至少 2 种内置技能、本地存储  
### 9.2 后续迭代  
- V1.1：多 Agent 角色扮演 + 初步 token 缓解  
- V1.2：MCP 集成 + 技能市场  
- V2.0：登录版、云同步、发布服务  
### 9.3 风险与依赖  
- 大模型 API 稳定性与成本控制，外部 MCP 服务的可用性，Token 消耗优化效果
